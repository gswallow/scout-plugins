Using Scout to Monitor Windows

  • Edit
  • Delete
  • Tags
  • Autopost

Problem

Monitoring a Windows server.

Background

As I mentioned before, I am responsible for interfacing with a process that has to run on a Windows server.

It has been a been a bit of a pain to monitor.  We are currently using Scout to monitor our Linux servers, and while I didn't see any indicator that Scout could monitor a Windows server, I didn't see any reason that it couldn't.  Clearly, using Scout to monitor this Windows box would make it easier to have data consolidated in one place, and we wanted to leverage the alerts and notifications in Scout to let us know when our Windows box may be in need of attention.

Solution

Scout works on Windows!  We have been using Scout to monitor our Windows server for a few months now, and it has worked flawlessly.

How To

Configure the Windows Server:

To run Scout on Windows, you will first need to have RubyGems installed and then install the Scout gem on the Windows server.  Installing the Scout gem works just like any other gem install.  After the install, there is one setup step to ensure that this server can report to your Scout account.

Now the Scout agent is installed on your Windows server.  You need to ensure that it is invoked regularly so that it can report data back to Scout.  Unfortunately, we don't have cron to use, and so we are left with "Scheduled Tasks".  You can find it here: Start -> All Programs -> Accessories -> System Tools -> Scheduled Tasks.  Create a new scheduled task (File -> New) named "Scout".  On the "Task" configuration tab under "Run:" you want to put the path to the scout agent followed by a space and then your scout key.  

Scheduling things in Windows to run more than once a day is not very intuitive.  You basically tell it to run one time a day and then set some advance options to tell it to run more frequently.

You need to set a few other things as well, here's what my tabs look like (I am using a fake key for an example below):

Now the Scout Agent should run every two minutes.  Note, it will fail because right out of the gate, it is going to try and do some Linux-style monitoring.

Configure the web application:

Log into scoutapp.com with your account and find your new server.

Delete all of the existing/default plugins for this server.  None of them will work.

Scout has a really cool and simple approach to adding your own plugins by pointing to scripts that are posted online (such as at github).  I have the ones that I wrote hosted here.  You need to use the "raw" format for Scout.  The README file at the bottom of the github page will give more information about each script.  Right now, I have scripts for CPU load, disk space, memory used, checking the file count in a directory, running a bat file, and checking to see if a given process/service is running or not.

Now we are ready to add our custom Windows monitoring plugins.  While looking at the server in Scout, hit "Add Plugin" and then use the "Create a plugin" link.  This will take you to a "Add a Plugin Manually" page.  This page will prompt you for a name and the URL to the plugin code.  Note, you can also copy/paste code directly into Scout, but I think that is a much harder way to manage the code.

For the first example, you can name this first plugin "Windows CPU Load" and for the URL to the code, enter http://github.com/djohnsonjr/scout-plugins/raw/master/windows_cpu_load/window... and then hit "Create Plugin".  Remember to use the "raw" format.  If you are navigating around Github, you will see links for "raw" and you can see in the URL that I provided that it contains the word "raw".

There is no additional configuration required to get it working!  Since we are reporting "% used" you can edit the "label" under the plugin's settings to have a "%" in the "units" field, but that's not required.

Some of the plugins that I have done do require additional configuration.  For example, if you wanted to use the plugin that counts files in directory, you have to set the directory as a configuration option when setup the plugin.  You would create the plugin the same as the CPU monitor.  This time name it "Input File Count" and give a URL of http://github.com/djohnsonjr/scout-plugins/raw/master/windows_check_for_files...

Once the plugin is added, the setting page for that plugin should have an additional option called "Directory" that wasn't on the last plugin.  In this directory box, put the full path of the Windows directory that you want to monitor, for example C:\input_files and update the settings.  Here, you may want to adjust the label settings and put "files" for the units.  This will make your dashboard say, "10 files" instead of just 10.

If you have two directories that you want to monitor, you simply add another plugin, and follow the same steps.

Finally, you can now set alerts and build charts against any of these monitors as you see fit.
